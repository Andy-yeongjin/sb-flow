사전 준비 (최초 1회)

프로젝트를 처음 시작할 때만 하면 됩니다. 이미 했으면 건너뛰세요.

프로젝트당 1번
1. Claude 데스크톱 앱 설치 ▼

Claude 데스크톱 앱을 아직 설치하지 않았다면 먼저 설치해야 합니다.

📖
Claude 데스크톱 앱 설치 가이드 →
설치가 이미 완료됐다면 다음 단계로 넘어가세요.
2. sb-flow 다운로드 ▼

GitHub 페이지 →에서 Code → Download ZIP을 클릭해서 다운로드합니다. 압축을 풀어서 원하는 위치에 저장해두세요.

💡
한 번만 받으면 됩니다. 이후 모든 프로젝트에서 재사용할 수 있습니다.
3. setup.bat 실행 — 프로젝트 폴더 자동 설정 ▼

sb-flow 폴더 안의 setup.bat을 더블클릭하면 폴더 선택 창이 뜹니다. 내 프로젝트를 만들 폴더를 선택하면 필요한 파일이 자동으로 복사됩니다.

💡
새 폴더를 만들어서 선택해도 됩니다. 빈 폴더여야 합니다.
4. Claude 데스크톱 앱에서 "코드" 탭 클릭 ▼

Claude 데스크톱 앱을 열고 상단 메뉴에서 "코드" 탭을 클릭합니다.

5. 프로젝트 폴더 열기 ▼

하단의 폴더 이름 옆 "지점 선택" 버튼을 클릭한 뒤, setup.bat으로 설정한 프로젝트 폴더를 선택합니다.

6. 권한 건너뛰기 설정 ▼

왼쪽 아래의 설정 버튼(⚙️)을 클릭합니다.

설정에서 "권한 우회 모드 허용" 토글을 켭니다.

입력창 왼쪽 아래의 권한 버튼을 클릭하고 "권한 건너뛰기"를 선택합니다. Claude가 매번 확인 없이 파일을 자유롭게 수정할 수 있게 됩니다.

7. /sb-setup 입력 — 도구 자동 설치 ▼

대화창에 아래를 입력하면 Claude가 Node.js 설치 확인부터 bkit, Vercel 스킬 등 필요한 도구를 자동으로 설치합니다.

Claude 대화창에 입력
/sb-setup
✅
설치 완료! 이제 아래 1단계로 넘어가서 /sb-oneshot을 실행하면 됩니다.

Claude에 붙여넣기 — 만들고 싶은 것을 자유롭게 설명
/sb-oneshot 여기에 만들고 싶은 기능을 설명하세요
✏️
예시로 보는 좋은 설명:
/sb-oneshot 포모도로 타이머. 25분 집중, 5분 휴식 사이클. 시작/정지 버튼, 오늘 완료 횟수 표시.
/sb-oneshot 식단 기록 앱. 음식 검색해서 끼니별 기록, 하루 칼로리 합계 표시.
/sb-oneshot 가계부. 수입/지출 입력, 카테고리별 분류, 월별 차트.
AI가 자동으로 실행하는 6단계
0/6 입력 자산 감지 (헌법·디자인 시스템·PRD·.pen/Stitch)
1/6 개발 계획 (/pdca plan)
2/6 상세 설계 (/pdca design)
3/6 코드 구현 (/pdca do)
4/6 갭 분석 + 디자인 토큰 검증
5/6 자동 개선 (Match Rate < 90% 시)
6/6 브라우저 기능·디자인 검증 + 자체 수정
⏱️
시간이 좀 걸립니다. 6단계를 순서대로 실행하기 때문에 10~30분 정도 기다리세요. 중간에 Claude가 질문하면 한국어로 답해주세요.
⚠️
중간에 멈추는 경우: 개발 서버가 실행 중이 아니면 Claude가 멈추고 서버를 켜달라고 안내합니다. start.bat으로 서버를 켠 후 "계속"을 입력하세요.
▶️
구현 완료! 브라우저에서 확인하려면:
프로젝트 폴더 안의 start.bat을 더블클릭하면 개발 서버가 시작되고 브라우저가 열립니다.
서버를 끄려면 end.bat을 더블클릭하세요.

자주 묻는 질문

{기능명}에 뭘 적어야 하나요? ▼
만들고 있는 기능의 짧은 이름을 적으세요. 예: 포모도로-타이머, 회원가입, 가계부. 한국어도 됩니다.
oneshot 중간에 에러가 나면? ▼
Claude가 안내 메시지를 줍니다. 대부분 질문에 답하면 해결됩니다. 막히면 /sb-guide를 실행해서 현재 상태를 확인하세요.
검증(3단계)을 꼭 해야 하나요? ▼
최소한 /pdca analyze와 /pdca report는 하는 게 좋습니다. 설계대로 만들어졌는지 확인하고, 기록을 남기는 거예요. 나머지는 건너뛰어도 됩니다.
전체 워크플로우 가이드는 없나요? ▼
sb-flow-workflow.html에 전체 6단계 워크플로우 가이드가 있습니다. /pdca 명령어를 단계별로 직접 제어하고 싶을 때 참고하세요.
